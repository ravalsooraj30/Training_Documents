	- Jenkins - Build Orchestration Tool :::
	
		- Is an open-source build orchestration Tool 
		- Used to Create the end-to-end CICD Pipeline to automation application builds and deployments
		- Used to perform any scheduled process automation 
	
	- Jenkins Architecture :: CI & CD 
	
		- Jenkins is based on master/Controller & slave/Node
		
			- Jenkins Master/Controller (VM) :		
								- Install Jenkins 
								- To Create Jenkins Jobs/Projects/Pipelines and scheduled to run on the Jenkins Slave Nodes 
		
				- Jenkins Slave_Node (VM) (Build Server):	
								- Install the Build Tools 
								- To Execute the Jobs  

	Alternative to Jenkins :
	
		- GitHub-Actions 
		- gitlab-ci 
		- Bamboo
		- Azure Pipelines 
		- AWS Code Pipelines 
	
	- Setup Jenkins Architecture :::

	Scenario1 :	Application Build and Release frequency is very less.
	
		Jenkins_Master (VM) 	
				==> Create Jenkins CI/CD Pipeline Projects & Execute the Build in same Jenkins_Master Node.		
		
		
		
	Scenario2 : Multiple Applications with high Build and Release frequencies.
		
		Jenkins_Master (VM)			
				==> To Create Jenkins CI/CD Pipeline Projects and schedule to the builds in the slave_nodes
																
			Jenkins_SlaveNode1 (VM)	==> Perform Application Builds - Java Applications 
			Jenkins_SlaveNode2 (VM)	==> Perform Application Builds - .Net Applications			
			Jenkins_SlaveNode3 (VM)	==> Perform Application Builds - Python Applications 
			Jenkins_SlaveNode4 (VM)	==> Perform Application Builds - Angular/NodeJS
			Jenkins_SlaveNode5 (VM)	==> Perform Application Builds - Ruby 		


	- Jenkins Pipelines/Jobs/Projects Workflow :
		
				- Application Build 
					==> Process of Compiling the Source Code, 
						Create the Artifacts(Executable code - Binaries - *.war/*.jar/*.exec/*.dll) 
				- Unit Test 
				- Promote the Changes to Test Environments
				- Notify the Testing Team thru Emails	
				
	CI/CD Workflow :::
	
		SCM_Checkout 
		Application Build 
		Unit Testing 
		Create Artifacts
		Promote the Application to Test Environments
		Notify 
		Deploy to Prod 	


	- Developers' Perspectives :
	
		- Developers are just the consumers of Jenkins using Jenkins Tools and Services.
		- Developers use Jenkins to execute/cancel/view the output - workspace of the Jenkins projects/Pipelines/Jobs.

	- DevOps Perspectives:
		
		- DevOps Team is the owner of Jenkins.
